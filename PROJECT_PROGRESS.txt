YOUTUBE VI SUBTITLES - PROJECT STATUS
=====================================

Updated: 13/06/2026

1. CURRENT GOAL
---------------

Chrome Manifest V3 extension that translates timestamped English YouTube
captions to Vietnamese. Translation is local-only through a FastAPI server using
vinai/vinai-translate-en2vi-v2. No Gemini, Claude, OpenAI or cloud API key is
used or stored.


2. CURRENT ARCHITECTURE
-----------------------

YouTube caption track / Transcript panel
  -> src/page-bridge.js obtains caption data
  -> src/subtitle-core.js parses timestamped cues
  -> src/content.js prioritizes current and upcoming cues
  -> background.js sends batches to http://127.0.0.1:8000/translate
  -> local-server/server.py runs VinAI on local CPU/GPU
  -> chrome.storage.local caches translated cues
  -> src/content.js renders the synchronized overlay in the video player

The model is downloaded from Hugging Face only on first use and stored in
local-server/.hf-cache/. Subsequent translation uses the local cache. The model
code/revision is pinned to commit:

82f8c91bd22e82085186b45a8a76373f5a79f667


3. IMPLEMENTED FEATURES
-----------------------

- JSON3, VTT, XML and YouTube Transcript-panel extraction.
- English manual-caption preference, then English auto-caption fallback.
- PO-token detection before retrying unusable direct caption URLs.
- Optional visible-CC realtime fallback when batch transcript loading fails.
- Batch translation in chunks of 8 cues.
- Pause/prefetch and resume after 16 upcoming cues are translated.
- **Xem ngay** action to skip waiting while translation continues.
- Request timeout of 30 seconds, two retries and active cancellation.
- Cache version 4 with partial-resume support.
- 30-day cache expiration, 25 MB limit and oldest-entry eviction.
- Debounced cache writes and manual cache clearing from the popup.
- Binary-search cue lookup instead of scanning every cue each render cycle.
- Vietnamese-only and bilingual English/Vietnamese display.
- Option to hide original YouTube CC and limit overlay line count.
- Font family, font size, preset text/background colors and opacity controls.
- Adjustable vertical position, direct drag and double-click position reset.
- Keyboard commands for translation toggle and font sizing.
- Automatic preservation of technical terms, acronyms, framework names, code
  identifiers, commands, URLs and filenames.
- Local logs that avoid storing subtitle text or credentials.
- Local server health check showing CPU/GPU and pinned model information.


4. SECURITY STATUS
------------------

- No cloud API keys, tokens, passwords or private keys are present in source.
- Manifest permissions are limited to storage, activeTab, YouTube and localhost.
- Legacy Gemini/Claude/OpenAI settings are deleted from chrome.storage.sync.
- Root .gitignore excludes .venv, .hf-cache, __pycache__, .env, key files,
  databases, logs and editor metadata.
- VinAI remote model code uses trust_remote_code=True with a pinned revision.
- FastAPI should remain bound to 127.0.0.1 and must not expose port 8000 publicly.
- Local requests are limited by MAX_BATCH_ITEMS and MAX_TEXT_LENGTH.

6. KNOWN LIMITATIONS
--------------------

- A video needs English captions, a usable Transcript panel or visible English
  CC. Speech-to-text for videos without captions is intentionally not included.
- YouTube may change internal player data, caption URLs or Transcript-panel DOM.
- Realtime visible-CC fallback has unavoidable translation latency.
- The first server start needs internet access to download the pinned model.
- New machines need their own Python environment, compatible PyTorch/CUDA setup
  and model cache.
- CPU translation works but is slower than CUDA.
- Full Chrome automation is not present; real YouTube videos still require
  manual validation.
- VinAI model licensing is AGPL-3.0 and must be reviewed before distribution or
  commercial use.


7. RELEASE READINESS
--------------------

The source tree is suitable for a new public GitHub repository after normal
manual Chrome validation. Do not upload local-server/.venv/,
local-server/.hf-cache/ or generated bytecode. These paths are already ignored.
